## 3. 文献分析测试

### 3.1 单篇文献分析

| 测试点 | 操作步骤 | 预期结果 |
|--------|----------|----------|
| 提交分析请求 | `POST /api/analysis/single` body: `{"fileName":"test.pdf","fileType":"pdf","text":"文献内容..."}` | 返回 SSE 流 |
| SSE 事件流 | 观察 SSE 事件 | 依次收到 `step` 事件（literature, relation, writing, todo, matching） |
| 分析完成 | 等待 SSE 流结束 | 收到 `complete` 事件，包含 `documentId` |
| 步骤状态 | 观察每个 step 事件 | 每个步骤先收到 `running` 状态，再收到 `done` 状态 |

### 3.2 前端集成测试

| 测试点 | 操作步骤 | 预期结果 |
|--------|----------|----------|
| 文件上传 | 在前端上传 PDF/TXT 文件 | 文件内容被提取并显示 |
| 分析进度 | 提交分析后观察进度条 | 进度条随步骤推进 |
| 分析结果 | 分析完成后查看结果 | 显示文献解析、关联分析、写作素材、待办提取、匹配评分 |

## 10. 完整流程测试

### 10.1 文献分析完整流程

1. **配置 LLM**
   - 设置 `llm_api_key`
   - 设置 `llm_base_url`
   - 设置 `llm_model`

2. **上传文献**
   - 选择 PDF/TXT 文件
   - 提交分析请求

3. **等待分析**
   - 观察进度条
   - 等待所有步骤完成

4. **查看结果**
   - 查看文献解析
   - 查看关联分析
   - 查看写作素材
   - 查看待办提取
   - 查看匹配评分

5. **管理知识库**
   - 查看文档列表
   - 删除文档

6. **查看历史**
   - 查看历史列表
   - 查看历史详情
